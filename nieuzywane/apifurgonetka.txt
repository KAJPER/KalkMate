Wersje językowe
Domyślnym językiem używanym w odpowiedziach we wszystkich endpointach jest język polski (pl_PL). Aby zmienić wersję językową należy dołączyć do żądania HTTP nagłówek X-Language używając jednej z dostępnych wartości:

en_GB
Brak przesłania nagłówka X-Language lub przesłanie innej wartości powoduje ustawienie wartości domyślnej pl_PL.
Limity zapytań
Ilość zapytań do API, które można wykonać jest ograniczona w czasie oraz może być bardziej restrykcyjna w zależności od rodzaju zapytania. Ograniczenia te pozwalają na zachowanie dostępności API dla wszystkich użytkowników. Zapytania nieuwierzytelnione są powiązane z adresem IP, z którego zostało wykonane żądanie.

Domyślne limity zapytań do API:
5000 na godzinę dla zapytań aplikacji oAuth
500 na godzinę dla zapytań nieuwierzytelnionych
Do każdej odpowiedzi z API dołączane są odpowiednie nagłówki informujące o obecnym stanie limitu zapytań. W przypadku przekroczenia limitu zapytań, API zwróci odpowiedź z kodem 429 Too Many Requests. W takiej styuacji zapytania do API nie powinny być ponawaiane przed upływem czasu wskazanego w nagłówku X-RateLimit-Retry-After.

Nagłówek	Opis
X-RateLimit-Limit	Maksymalna liczba zapytań do API, która może być wykonana w ciągu godziny.
X-RateLimit-Remaining	Pozostała liczba zapytań do API, która może zostać wykonana w obecnym oknie czasowym.
X-RateLimit-Retry-After	Czas w sekundach, po którym obecne okno czasowe zostanie zresetowane. W przypadku kiedy limit zapytań nie został przekroczony, wartość nagłówka będzie wynosiła 0.
